# Origin Client — 1.21.11 port status & guide

1.21.11 is a **Tier A modern** port (Fabric, official Mojang mappings, Java 21).
It is the 1.21.1 module with exactly one version-forced change: the
**`GuiGraphics.blit` rework Minecraft introduced in 1.21.2**. Everything else —
theme tokens, baked assets, the mods registry/config, HUD layout math, and the
entire mixin layer — is byte-identical to `OriginClient.Mod` (1.21.1), so the
look and feel match 1.21.1 exactly (the product goal for this version).

## Install model
Matches 1.20.4 / 26.2, NOT 1.21.1: the built jar carries **only** the Origin
client. The launcher installs the matching 1.21.11 perf/shader stack standalone
from `PerformanceModCatalog` (`VersionManager.OriginBuilds` records
`BundlesPerfStack: false`). The 1.21.11 catalog entry is already `Full`
(Sodium `mc1.21.11-0.8.13`, Iris `1.10.4+1.21.11`, Lithium, FerriteCore,
Krypton), so shaders were already available for 1.21.11 as a vanilla-menu
Fabric version — this module adds the Origin menus on top.

## The one API delta: the 1.21.2 blit rework (DONE, javap-verify pending)
Real Minecraft reworked `GuiGraphics.blit` at 1.21.2: a
`Function<ResourceLocation, RenderType>` (use `RenderType::guiTextured`) is
prepended, and the region-scaled overload moves the u/v offsets **ahead of** the
destination width/height. Nothing else in the GUI draw path changed at 1.21.2 —
`RenderSystem.setShaderColor` tinting, `enableBlend`/`defaultBlendFunc`, and
`GuiGraphics.pose()` (`PoseStack`) are all unchanged from 1.21.1 (that removal is
a later, 26.x change — see `../OriginClient.Mod262/PORT-262.md`; do not apply it
here).

Rewrite rules applied across the five drawing files:

- **Full-texture form**
  `blit(id, x, y, u, v, w, h, texW, texH)`
  → `blit(RenderType::guiTextured, id, x, y, u, v, w, h, texW, texH)` (prepend only)
- **Region-scaled form**
  `blit(id, x, y, dstW, dstH, u, v, srcW, srcH, texW, texH)`
  → `blit(RenderType::guiTextured, id, x, y, u, v, dstW, dstH, srcW, srcH, texW, texH)`
  (prepend + move u/v ahead of dstW/dstH; int u/v cast to float)
- **Atlas-sprite form** (HudElements potion icons)
  `blit(x, y, z, w, h, TextureAtlasSprite)`
  → `blitSprite(RenderType::guiTextured, sprite, x, y, w, h)` (least-certain
  descriptor; already fail-soft via try/catch)

Files touched (33 blit call sites, all now `RenderType::guiTextured`):
`gui/OriginUi.java`, `gui/OriginButtonRenderer.java`,
`render/OriginScreenRenderer.java`, `hud/HudElements.java`,
`shaders/ShaderBrowserScreen.java`. `mods/BlockOverlayRenderer.java` and
`mods/ChunkBorderRenderer.java` draw world-space lines via `RenderType.lines()`
(not GUI blit) and are unchanged.

## Verify at home before flipping live (repo discipline — CLAUDE.md)
The sandbox has no Mojang/Loom access, so this could not be compiled here (same
constraint every prior port documented). On a machine with build access:

1. Confirm coordinates on https://fabricmc.net/develop for 1.21.11 and fix
   `gradle.properties` (`minecraft_version`, `fabric_api_version`) if they drift.
2. `./gradlew build` — resolves + remaps MC 1.21.11 and compiles. The reference
   jar for javap is `~/.gradle/caches/fabric-loom/1.21.11/minecraft-client.jar`.
3. **javap every rewritten `blit`/`blitSprite` descriptor** against that jar —
   especially the region-scaled arg order and the `blitSprite(RenderType,
   TextureAtlasSprite, ...)` overload — this is the whole risk surface of the
   port.
4. Drop the dev-only `libs/sodium-fabric-0.8.13+mc1.21.11.jar` +
   `libs/iris-fabric-1.10.4+mc1.21.11.jar` (URLs in the catalog's 1.21.11 entry)
   and `./gradlew runClient`: confirm **zero `Mixin apply ... failed`** lines,
   the title/loading/mod-menu/HUD all render Origin (identical to 1.21.1), and a
   shader pack applies (Iris integration).
5. Flip it live: uncomment the `["1.21.11"]` line in
   `VersionManager.OriginBuilds`, re-enable the build steps in
   `build-check.yml` + `launcher-release.yml`, bump the csproj jar filename if
   `mod_version` changed. The csproj Content item and the catalog entry are
   already in place.
